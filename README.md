# TaskFlow

Веб-приложение для управления личными задачами с возможностью категоризации, фильтрации и уведомлений.

## Цель проекта
Снизить когнитивную нагрузку при планировании ежедневных задач за счёт централизованного хранения, быстрой смены статусов и автоматических напоминаний. Целевая аудитория — пользователи, предпочитающие лёгкие инструменты вместо комплексных систем управления проектами.

## Основной функционал
- Регистрация и авторизация по email/паролю
- Создание, редактирование и удаление задач
- Управление статусами: `новая`, `в работе`, `выполнена`, `отложена`
- Добавление тегов и фильтрация по дедлайнам
- Отправка email-уведомлений о приближении дедлайна
- Адаптивный интерфейс для мобильных устройств

## Архитектура и технологический стек
| Компонент       | Технология                          | Примечание                                      |
|-----------------|-------------------------------------|-------------------------------------------------|
| Фронтенд        | React + TypeScript + Vite           | SPA, роутинг через React Router                 |
| Управление состоянием | Zustand (UI) + TanStack Query (API) | Разделение локального и серверного состояния    |
| Бэкенд          | Node.js + Express                   | REST API, аутентификация JWT                    |
| База данных     | PostgreSQL + Prisma ORM             | Схема данных с отношениями 1:N (User → Tasks)   |
| Валидация       | Zod                                 | Проверка входящих данных на уровне маршрутов    |
| Инфраструктура  | Docker, GitHub Actions, Railway     | Контейнеризация, автоматический деплой          |

Архитектура построена по слоистому принципу: маршруты → контроллеры → сервисы → репозитории. Клиент взаимодействует с сервером через REST API с пагинацией и стандартными HTTP-кодами состояний.

## План разработки
1. Проектирование схемы БД и прототипирование интерфейса
2. Настройка окружения, инициализация бэкенда (аутентификация, базовые маршруты)
3. Реализация сервисного слоя и подключения Prisma
4. Разработка фронтенда: роутинг, формы, интеграция с API
5. Написание unit-тестов (бэкенд) и integration-тестов (API)
6. Настройка CI/CD через GitHub Actions, деплой на Railway/Vercel
7. Финальное тестирование и исправление дефектов
